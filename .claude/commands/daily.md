# /daily — 오늘 배운 것 기록

오늘 하루 배운 것, 만든 것, 막힌 것을 5개 질문으로 정리합니다.

## 사용법

```
/daily
```

---

## 실행 순서

## 공통 실행 원칙

- 학생에게 터미널 명령을 직접 입력하라고 시키지 마.
- 최근 커밋 확인, 폴더 생성, 파일 저장은 Claude가 직접 실행해.
- 학생에게는 질문에 답하게 하고, 저장된 경로와 다음 선택지만 알려줘.

---

### 1단계: 오늘 컨텍스트 파악

학생에게 명령어를 치라고 하지 말고, Claude가 직접 `git log --oneline -5`로 최근 커밋 내역을 확인해줘.

- 커밋이 있으면 → 파악한 내용을 한 줄로 보여주고 맞는지 확인해:
  ```
  오늘 [확인한 내용] 작업하셨군요! 맞나요?
  ```
- 커밋이 하나도 없으면 (에러가 나거나 결과가 비어 있으면) → 이렇게 말해:
  ```
  오늘이 첫 시작이군요! 어떤 걸 해보셨나요?
  ```

---

### 2단계: 5개 질문 (하나씩 순서대로)

**Q1. 오늘 한 것**
```
Q1/5 — 오늘 뭘 만들거나 시도했나요?
```

**Q2. 잘 된 것**
```
Q2/5 — 뭐가 잘 됐나요? 예상보다 쉬웠던 것도 좋아요.
```

**Q3. 막힌 것**
```
Q3/5 — 막히거나 이해가 안 됐던 게 있었나요?
(없으면 "없어"라고 해도 돼요)
```

**Q4. 새로 배운 것**
```
Q4/5 — 오늘 새로 알게 된 것이 있나요?
예: 코드 개념, 도구 사용법, 문제 해결 방법 ...
```

**Q5. 내일 할 것**
```
Q5/5 — 내일 해보고 싶은 것이 있나요?
```

---

### 3단계: 기록 저장

**저장 경로**: `notes/daily/YYYY-MM-DD.md`

파일이 이미 있으면 → 오늘 내용을 아래에 이어서 추가 (덮어쓰지 마)  
파일이 없으면 → 새로 생성

**파일 내용:**

```markdown
---
date: YYYY-MM-DD
---

# YYYY-MM-DD 배움 기록

## 오늘 한 것
(Q1 답변)

## 잘 된 것
(Q2 답변)

## 막힌 것
(Q3 답변, 없으면 생략)

## 새로 배운 것
(Q4 답변)

## 내일 할 것
(Q5 답변)
```

---

### 4단계: 마무리

저장 후 이렇게 말해줘:

```
오늘 하루 수고했어요! 📝
기록을 notes/daily/YYYY-MM-DD.md에 저장했어요.

막힌 게 있었다면 지금 같이 해결해볼까요?
```

Q3에 막힌 내용이 있으면 → 해결 제안을 먼저 해줘도 좋아.

---

## 주의사항

- 짧게 답해도 괜찮아. 한 단어도 OK
- 오늘 아무것도 못 했어도 괜찮아. 그것도 기록이야
- 너무 잘 써야 한다는 부담 주지 마
