# /start — 첫 시작 온보딩

처음 이 워크스페이스를 열었을 때 실행하는 커맨드입니다.
학생에게 몇 가지 질문을 하고, 그 답변을 바탕으로 워크스페이스를 개인화합니다.

---

## 실행 순서

### 1단계: 환영 인사

다음 메시지를 출력해줘:

```
안녕하세요! 바이브코딩 워크스페이스에 오신 걸 환영해요 👋

저는 여러분의 AI 코딩 파트너 Claude예요.
몇 가지 질문에 답해주시면, 여러분에게 딱 맞는 첫 번째 자동화 프로젝트를 함께 시작할게요.

총 6개 질문이에요. 짧게 답해도 좋아요! (나머지 설정은 자동으로 해줄게요)
```

---

### 2단계: 질문지 (순서대로, 하나씩)

질문을 **하나씩 순서대로** 물어봐줘. 한꺼번에 다 물어보지 말고, 답변 받으면 다음 질문으로 넘어가.

**Q1. 이름**
```
Q1/6 — 어떻게 불러드릴까요? (닉네임도 좋아요)
```

**Q2. 전공/관심 분야**
```
Q2/6 — 전공이나 요즘 가장 관심 있는 분야가 뭐예요?
예: 데이터과학, 경영, 디자인, 영상편집, 게임 ...
```

**Q3. 가장 시간을 많이 쓰는 반복 작업**
```
Q3/6 — 공부하거나 일할 때, 반복적으로 하는 귀찮은 작업이 있나요?
예: 매주 엑셀 정리, 논문 PDF 정리, 유튜브 자막 노트 정리, 다운로드 폴더 지저분함 ...
```

**Q4. 자동화하고 싶은 것**
```
Q4/6 — 자동화할 수 있다면 뭘 제일 먼저 해보고 싶어요?
(아직 모르겠으면 "모르겠어"라고 해도 괜찮아요)
```

**Q5. 바이브코딩을 배우려는 이유**
```
Q5/6 — 이걸 배우는 이유가 뭐예요?
예: 공모전 준비, 취업 포트폴리오, 그냥 궁금해서, 교수님이 시켜서 ...
```

**Q6. 현재 코딩 실력**
```
Q6/6 — 지금 코딩 실력은 어느 정도예요?

1) 완전 처음 — 코드가 뭔지도 잘 모름
2) 기초 — 파이썬 조금 해봄, 변수·반복문 정도는 앎
3) 중급 — 혼자 작은 프로그램 만들 수 있음
```

---

### 3단계: Git 사용자 정보 확인

`git config --global user.name`과 `git config --global user.email`을 확인해줘.

- **이름이나 이메일이 비어 있으면** → 조용히 Q1 답변(이름)과 아래 형식으로 설정해:
  ```
  git config --global user.name "[Q1 답변]"
  git config --global user.email "[Q1닉네임]@vibe.local"
  ```
  설정 후 이렇게 알려줘:
  ```
  (Git 사용자 정보도 자동으로 설정해뒀어요!)
  ```
- **이미 설정되어 있으면** → 그냥 넘어가.

---

### 5단계: 답변 분석 및 CLAUDE.md 업데이트

모든 답변을 받은 뒤, `CLAUDE.md`의 `## 나에 대해` 섹션을 아래 내용으로 교체해줘:

```markdown
## 나에 대해

- 이름: [Q1 답변]
- 전공/관심 분야: [Q2 답변]
- 자주 하는 반복 작업: [Q3 답변]
- 자동화하고 싶은 것: [Q4 답변]
- 바이브코딩 목표: [Q5 답변]
- 코딩 실력: [Q6 답변 기반으로 "입문" / "기초" / "중급" 중 하나]
```

---

### 6단계: 첫 번째 프로젝트 추천

Q3, Q4 답변을 기반으로 **첫 번째 프로젝트**를 추천해줘.

추천 기준:
- Q3, Q4에서 언급된 작업이 있으면 → 그것과 연관된 자동화
- "모르겠어" 또는 아무것도 없으면 → 아래 기본 추천 중 하나

**기본 추천 목록** (코딩 실력에 맞게 선택):
- 입문: 다운로드 폴더 파일 정리 자동화 (파이썬 기초)
- 기초: 엑셀/CSV 데이터 자동 정리 스크립트
- 중급: 웹에서 원하는 정보 자동 수집 스크립트

추천 메시지 형식:
```
[이름]님 답변 잘 받았어요!

[Q3/Q4 답변 기반 공감 한 마디]

첫 번째 프로젝트로 이걸 추천해요:

📦 프로젝트명: [프로젝트 이름]
📝 설명: [한 두 문장 설명]
⏱ 예상 시간: [30분 / 1시간 / 2시간 중 하나]

지금 바로 시작할까요? (네 / 아니오)
```

---

### 7단계: 프로젝트 시작 (동의 시)

"네" 또는 동의하는 답변이 오면:

1. `.claude/references/project-readme-guide.md`를 먼저 읽어
2. `projects/[프로젝트명]/` 폴더 생성
3. 아래 기본 폴더 생성:
   - `src/` — 코드
   - `input/` — 입력 파일
   - `output/` — 결과 파일
   - `notes/` — 생각/실험 기록
4. `projects/[프로젝트명]/README.md` 생성

README는 절대 간단히 쓰지 말고, 학생이 다음 날 다시 열어도 바로 이어서 할 수 있는 실행 계획서로 작성해.

**필수 섹션:**

````markdown
# [프로젝트명]

## 한 줄 목표
[무엇을 입력하면 무엇이 만들어지는지 한 문장]

## 왜 이걸 만들까?
- [Q3/Q4 답변에서 나온 불편함]
- [완성되면 좋아지는 점]

## 최종 결과물
- [생성될 파일 또는 화면]
- [성공하면 학생이 확인할 수 있는 결과]

## 사용 시나리오
1. [사용자가 준비할 것]
2. [실행 방법]
3. [결과 확인 방법]

## 입력
- [필요한 파일/텍스트/폴더/URL]

## 출력
- [생성되는 파일명/폴더명]

## 필요한 도구
- Python
- [필요한 라이브러리나 도구]

## 폴더 구조
```text
[프로젝트명]/
├── README.md
├── src/
├── input/
├── output/
└── notes/
```

## 만들 기능
- [ ] 1단계: 최소 버전 만들기
- [ ] 2단계: 테스트 데이터로 실행하기
- [ ] 3단계: 결과 파일 저장하기
- [ ] 4단계: 오류 처리하기
- [ ] 5단계: 사용법 정리하기

## 첫 번째 작업
[지금 바로 할 수 있는 가장 작은 행동 1개]

## Claude에게 다음에 요청할 말
```text
[학생이 그대로 복사해서 보낼 수 있는 다음 요청]
```

## 메모

````

5. `.claude/references/project-quality-checklist.md`로 빠진 항목이 없는지 점검해
6. README를 만든 뒤 첫 번째 작업을 단계별로 안내해줘

**중요:** 프로젝트가 `강의 PPT 자동 생성기`, `PPT 만들기`, `발표자료 자동화`처럼 발표자료와 관련 있으면 `python-pptx` 기반 예시를 참고해서 입력/출력/폴더 구조를 구체적으로 써줘.

## 주의사항

- 질문은 반드시 하나씩, 친근하게
- 전문 용어 사용 금지 (초보 대상)
- 답변이 짧거나 엉뚱해도 당황하지 말고 맞춰서 진행해줘
- "모르겠어", "없어" 같은 답변도 정상 처리해줘
- `/start`를 이미 실행한 적 있으면 (CLAUDE.md에 이름이 있으면) "이미 설정됐어요! 다시 하시겠어요?" 물어봐줘
