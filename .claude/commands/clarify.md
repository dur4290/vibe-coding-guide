# /clarify — 자동화 아이디어 → 실행 계획서

막연한 자동화 아이디어를 구체적인 실행 계획서로 바꿔줍니다.

## 사용법

```
/clarify 인스타그램 댓글 정리하는 거 자동화하고 싶어
/clarify 매일 학교 공지사항 확인하는 게 너무 귀찮아
/clarify 다운로드 폴더가 너무 복잡해
```

인자 없이 `/clarify`만 써도 됩니다. Claude가 먼저 물어봐요.

---

## 실행 순서

### 1단계: 아이디어 파악

인자가 있으면 → 그 내용으로 시작  
인자가 없으면 → 이렇게 물어봐:

```
어떤 걸 자동화해보고 싶어요? 귀찮거나 반복적인 일이 있으면 말해줘요.
```

---

### 2단계: 6개 질문 (하나씩 순서대로)

**Q1. 빈도**
```
Q1/6 — 이 작업을 얼마나 자주 해요?
예: 매일, 주 3번, 시험 기간에만 ...
```

**Q2. 시간**
```
Q2/6 — 한 번 하는 데 얼마나 걸려요?
예: 10분, 30분, 1시간 ...
```
→ 답변 받으면 월간/연간 절약 시간을 자동 계산해서 보여줘:
> "월에 약 X시간, 1년이면 약 Y시간 절약할 수 있어요!"

**Q3. 가장 귀찮은 부분**
```
Q3/6 — 이 과정에서 가장 귀찮거나 실수하기 쉬운 부분이 뭐예요?
```

**Q4. 데이터 출처**
```
Q4/6 — 필요한 정보나 데이터를 어디서 가져와요?
예: 엑셀 파일, 특정 웹사이트, 카카오톡, 이메일 ...
```

**Q5. 원하는 결과물**
```
Q5/6 — 자동화가 되면 결과를 어떻게 받고 싶어요?
예: 파일로 저장, 카톡 알림, 화면에 출력, 이메일 ...
```

**Q6. 성공 기준**
```
Q6/6 — 자동화가 성공했다고 느끼는 순간이 언제일 것 같아요?
예: 버튼 하나로 정리가 끝날 때, 매일 알림이 올 때 ...
```

---

### 3단계: 실행 계획서 생성 및 저장

계획서를 만들기 전에 `.claude/references/project-readme-guide.md`를 참고해.
결과물은 짧은 메모가 아니라, 나중에 실제 프로젝트 README로 옮겨도 될 만큼 구체적으로 작성해.

모든 답변을 바탕으로 아래 형식의 파일을 생성해:

**저장 경로**: `notes/tasks/YYYYMMDD-[자동화이름].md`
- 날짜는 오늘 날짜 (KST)
- 자동화이름: 2-4 단어 요약, 한글 가능, 공백은 하이픈

**파일 내용:**

````markdown
---
date: YYYY-MM-DD
status: 계획중
---

# [자동화 이름]

## 한 줄 요약
(무엇을 자동화하는지 한 문장으로)

## 왜 자동화하고 싶나요?
(Q3 답변 기반)

## 얼마나 절약되나요?
- 빈도: (Q1)
- 1회 소요 시간: (Q2)
- 월 절약: 약 X시간 / 연 절약: 약 Y시간

## 어떻게 동작하나요?
1. (데이터를 Q4에서 가져옴)
2. (처리 과정)
3. (Q5 형태로 결과 출력)

## 입력
- (필요한 파일, 폴더, 텍스트, URL 등)

## 출력
- (생성될 파일, 화면 출력, 알림 등)

## 성공 기준
(Q6 답변)

## 기술 방향 (Claude 판단)
- 난이도: 쉬움 / 보통 / 어려움
- 필요한 것: Python, 특정 라이브러리 등
- 예상 소요 시간: X시간

## 예상 폴더 구조
```text
프로젝트이름/
├── README.md
├── src/
├── input/
├── output/
└── notes/
```

## 최소 버전
처음에는 너무 크게 만들지 말고 아래까지만 구현:
1. (가장 작은 입력)
2. (가장 작은 처리)
3. (눈으로 확인 가능한 결과)

## 나중에 개선할 것
- (추가 기능 1)
- (추가 기능 2)
- (오류 처리 또는 편의 기능)

## 진행 상황
- [ ] 계획 수립
- [ ] 첫 버전 완성
- [ ] 테스트
- [ ] 개선
````

---

### 4단계: 시작 제안

파일 저장 후 이렇게 물어봐:

```
계획서를 notes/tasks/에 저장했어요!

이제 바로 만들어볼까요?
"만들어줘" 라고 하면 지금 시작할게요. 
나중에 하고 싶으면 이 파일을 열어서 "이거 만들어줘" 라고 하면 돼요.
```

사용자가 "만들어줘"라고 하면:
1. `projects/[자동화이름]/` 폴더 생성
2. `src/`, `input/`, `output/`, `notes/` 폴더 생성
3. 계획서 내용을 바탕으로 풍부한 `README.md` 생성
4. `.claude/references/project-quality-checklist.md`로 빠진 항목 점검
5. 첫 번째 최소 버전부터 구현

---

## 주의사항

- 질문은 반드시 하나씩, 친근하게
- 기술 용어 사용 금지
- "모르겠어"도 괜찮다고 하고 함께 생각해줘
- 계획서에 너무 어려운 기술은 적지 마. 학생이 할 수 있는 수준으로
