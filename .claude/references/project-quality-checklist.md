# 프로젝트 생성 품질 체크리스트

새 프로젝트 폴더나 README를 만든 뒤 아래를 스스로 확인합니다.

## README 체크

- [ ] 한 줄 목표가 구체적인가?
- [ ] 왜 만드는지 사용자의 불편함과 연결되어 있는가?
- [ ] 입력이 무엇인지 적혀 있는가?
- [ ] 출력이 무엇인지 적혀 있는가?
- [ ] 실행 후 성공을 어떻게 확인하는지 적혀 있는가?
- [ ] 폴더 구조가 적혀 있는가?
- [ ] 최소 버전과 나중에 개선할 것이 분리되어 있는가?
- [ ] 학생이 그대로 복사할 수 있는 다음 요청문이 있는가?

## 폴더 체크

- [ ] `src/`가 있는가?
- [ ] `input/`이 있는가?
- [ ] `output/`이 있는가?
- [ ] `notes/`가 있는가?

## 초보자 친화성 체크

- [ ] 어려운 용어를 그대로 두지 않았는가?
- [ ] 첫 번째 작업이 5분 안에 가능한가?
- [ ] 코드 작성 전 입력/출력부터 정리했는가?
- [ ] 다음 단계가 한 문장으로 분명한가?

## 부족하면 보완

체크가 2개 이상 비면 README를 다시 보완합니다.
학생에게 "완성했어요"라고 말하기 전에 빠진 부분을 채웁니다.
