# 프로젝트 README 작성 기준

새 프로젝트를 만들 때는 `projects/프로젝트명/README.md`를 절대 짧게 만들지 않습니다.
README는 학생이 다음 날 다시 열어도 바로 이어서 작업할 수 있는 실행 계획서여야 합니다.

## 반드시 포함할 섹션

````markdown
# [프로젝트명]

## 한 줄 목표
[무엇을 입력하면 무엇이 만들어지는지 한 문장]

## 왜 이걸 만들까?
- [사용자가 불편하다고 말한 점]
- [완성되면 좋아지는 점]

## 최종 결과물
- [생성될 파일 또는 화면]
- [사용자가 확인할 수 있는 성공 결과]

## 사용 시나리오
1. [사용자가 준비할 것]
2. [스크립트/도구를 실행하는 방법]
3. [결과물을 확인하는 방법]

## 입력
- [필요한 파일, 텍스트, 폴더, URL]

## 출력
- [생성되는 파일명/폴더명]

## 필요한 도구
- Python
- [필요한 라이브러리]

## 폴더 구조
```text
프로젝트명/
├── README.md
├── src/
├── input/
├── output/
└── notes/
```

## 만들 기능
- [ ] 1단계: 최소 버전 만들기
- [ ] 2단계: 테스트 데이터로 실행하기
- [ ] 3단계: 결과 파일 저장하기
- [ ] 4단계: 오류 처리하기
- [ ] 5단계: 사용법 정리하기

## 첫 번째 작업
[지금 바로 할 수 있는 가장 작은 행동 1개]

## Claude에게 다음에 요청할 말
```text

```
````

## 작성 원칙

- 초보자가 파일을 닫았다가 다시 열어도 이해되게 씁니다.
- 코드부터 만들지 말고, 입력/출력/성공 기준을 먼저 정리합니다.
- 프로젝트 이름만 보고 내용을 추측하지 말고 사용자의 답변을 반영합니다.
- 어려운 기술명은 최소화하고, 필요한 경우 괄호로 쉽게 설명합니다.
- `첫 번째 작업`은 5분 안에 할 수 있는 행동으로 씁니다.
- `Claude에게 다음에 요청할 말`은 학생이 그대로 복사할 수 있게 작성합니다.

## 예시: 강의 PPT 자동 생성기

````markdown
# 강의 PPT 자동 생성기

## 한 줄 목표
강의 주제와 목차를 입력하면 기본 구조가 잡힌 PPT 파일을 자동으로 만들어줍니다.

## 왜 이걸 만들까?
- 매번 PPT 제목 슬라이드, 목차, 본문 슬라이드를 반복해서 만드는 시간이 아깝습니다.
- 초안을 빨리 만들고 내용 다듬기에 시간을 더 쓰기 위해서입니다.

## 최종 결과물
- `output/강의자료.pptx`
- 제목 슬라이드, 목차 슬라이드, 본문 슬라이드가 포함된 PPT

## 사용 시나리오
1. `input/topic.txt`에 강의 주제와 목차를 적습니다.
2. Python 스크립트를 실행합니다.
3. `output/` 폴더에 생성된 PPT를 열어 확인합니다.

## 입력
- 강의 제목
- 강의 대상
- 목차 또는 핵심 키워드
- 슬라이드 개수

## 출력
- `.pptx` 파일

## 필요한 도구
- Python
- python-pptx

## 폴더 구조
```text
강의PPT자동생성기/
├── README.md
├── src/
│   └── make_ppt.py
├── input/
│   └── topic.txt
├── output/
└── notes/
```

## 만들 기능
- [ ] 입력 파일에서 강의 제목과 목차 읽기
- [ ] 제목 슬라이드 만들기
- [ ] 목차 슬라이드 만들기
- [ ] 목차별 본문 슬라이드 만들기
- [ ] `output/` 폴더에 PPT 저장하기
- [ ] 파일이 없을 때 친절한 오류 메시지 보여주기

## 첫 번째 작업
`input/topic.txt` 파일을 만들고 강의 제목, 대상, 목차 예시를 적습니다.

## Claude에게 다음에 요청할 말
```text
강의 PPT 자동 생성기 프로젝트를 시작할게.
먼저 input/topic.txt 예시 파일과 src/make_ppt.py 첫 버전을 만들어줘.
python-pptx를 사용하고, 각 줄에 초보자용 주석을 달아줘.
```
````
