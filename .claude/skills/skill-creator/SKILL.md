# 스킬 만들기 가이드

새 스킬을 만들 때 이 가이드를 따라주세요.

## 스킬이란?

자주 쓰는 기능을 묶어서 재사용할 수 있게 만든 패키지입니다.  
`"XXX 스킬 써줘"` 라고 하면 Claude가 SKILL.md를 읽고 그 방식대로 작동해요.

---

## 스킬 구조

```
.claude/skills/스킬이름/
├── SKILL.md          ← 필수: 이 스킬이 뭘 하는지 설명
└── scripts/          ← 선택: 실행할 Python 스크립트
    └── 스크립트.py
```

---

## SKILL.md 형식

```markdown
---
name: 스킬이름
description: 한 줄 설명 (언제 이 스킬을 쓰는지)
allowed-tools: Read, Write, Bash, Glob
---

# 스킬 제목

## 트리거 문구
(학생이 어떻게 말하면 이 스킬을 쓰는지)

## 이 스킬이 하는 일
(구체적인 동작 설명)

## 사용법
(예시 포함)

## 저장 위치
(결과물을 어디에 저장하는지)

## 주의사항
```

---

## 새 스킬 만드는 순서

사용자가 "XXX 스킬 만들어줘" 라고 하면:

### 1단계: 스킬 정의 확인

아래를 먼저 물어봐 (한번에):
```
스킬을 만들기 전에 몇 가지 확인할게요:

1. 이 스킬은 어떤 상황에서 쓸 건가요?
2. 결과물을 어디에 저장하면 좋을까요?
3. 파이썬 스크립트가 필요한가요, 아니면 Claude가 직접 처리하면 되나요?
```

### 2단계: 기존 스킬 확인

```bash
ls .claude/skills/
```
비슷한 스킬이 이미 있으면 알려주고 확장할지 새로 만들지 물어봐.

### 3단계: 스킬 생성

- 폴더: `.claude/skills/스킬이름/` (소문자 + 하이픈)
- `SKILL.md` 작성 (500줄 이하로 유지)
- Python 스크립트가 필요하면 `scripts/` 폴더에 생성

### 4단계: 완료 알림

```
✅ '[스킬이름]' 스킬을 만들었어요!

위치: .claude/skills/스킬이름/SKILL.md
사용 방법: "[트리거 문구]" 라고 하면 이 스킬이 동작해요.

지금 바로 테스트해볼까요?
```

---

## 좋은 스킬의 특징

- SKILL.md는 간결하게 (500줄 이하)
- 트리거 문구가 명확함 (언제 이 스킬을 쓰는지 바로 앎)
- 저장 경로가 명확함
- 학생 수준에 맞는 언어 사용
