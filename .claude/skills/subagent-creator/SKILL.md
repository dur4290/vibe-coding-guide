# 서브 에이전트 만들기

새 에이전트를 만들 때 이 가이드를 따라주세요.

## 에이전트란?

특정 상황이 되면 Claude가 자동으로 불러서 일을 맡기는 전문가입니다.  
예: 코드가 만들어지면 → `code-simplifier`가 자동으로 주석을 달아줌

슬래시 커맨드와 다른 점: **사용자가 직접 부르지 않아도** 적절한 상황에서 자동으로 동작해요.

---

## 에이전트 파일 형식

```markdown
---
name: 에이전트이름
description: 언제 이 에이전트를 쓰는지 설명. "proactively"를 포함하면 자동 실행됨.
model: sonnet  ← sonnet / opus / haiku 중 선택
tools: Read, Write, Edit, Bash  ← 필요한 도구만
---

# 에이전트 제목

## 역할
(이 에이전트가 하는 일)

## 실행 조건
(어떤 상황에서 동작하는지)

## 처리 순서
(단계별 지시)

## 절대 하지 않는 것
(제약사항)
```

---

## 모델 선택 기준

| 모델 | 사용 시점 |
|------|---------|
| `haiku` | 단순 작업 (파일 읽기, 간단한 변환) |
| `sonnet` | 일반 작업 (기본값, 대부분 여기) |
| `opus` | 복잡한 분석, 고품질 글쓰기 |

---

## 새 에이전트 만드는 순서

사용자가 "XXX 에이전트 만들어줘"라고 하면:

### 1단계: 목적 확인

한번에 물어봐:
```
에이전트를 만들기 전에 확인할게요:

1. 이 에이전트는 어떤 상황에서 자동으로 동작하면 좋을까요?
2. 에이전트가 파일을 수정해야 하나요, 읽기만 하면 되나요?
3. 사용자 확인 없이 자동으로 실행해도 괜찮나요?
```

### 2단계: 기존 에이전트 확인

```
ls .claude/agents/
```
비슷한 에이전트가 있으면 알려줘.

### 3단계: 에이전트 파일 생성

저장 위치: `.claude/agents/에이전트이름.md`
- 이름: 소문자 + 하이픈 (예: `test-checker.md`)
- description에 "proactively" 포함 시 자동 실행됨
- 파일 수정 권한이 필요하면 tools에 `Edit`, `Write` 포함

### 4단계: 완료 알림

```
✅ '[에이전트이름]' 에이전트를 만들었어요!

동작 조건: [설명]
파일 위치: .claude/agents/에이전트이름.md

지금 테스트해볼까요?
```

---

## 주의사항

- 에이전트는 **사용자 확인 없이 파일을 수정**할 수 있어요. 범위를 좁게 설정하는 게 안전해요.
- description이 명확할수록 적절한 타이밍에 잘 실행돼요.
- 처음엔 `haiku`나 `sonnet`으로 시작하고, 필요하면 `opus`로 올려요.
