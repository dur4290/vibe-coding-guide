# 슬래시 커맨드 만들기

새 `/커맨드`를 만들 때 이 가이드를 따라주세요.

## 슬래시 커맨드란?

`/커맨드이름`을 입력하면 Claude가 `.claude/commands/커맨드이름.md`를 읽고 그 지시대로 동작합니다.  
반복적으로 하는 작업을 커맨드로 만들어두면 매번 긴 설명을 안 해도 돼요.

---

## 커맨드 파일 형식

```markdown
---
description: 한 줄 설명 (커맨드 목록에서 보임)
argument-hint: <인자 설명>  ← 인자가 있을 때만
---

# /커맨드이름 — 제목

## 사용법
(예시 포함)

## 실행 순서
(Claude가 따라야 할 단계별 지시)

## 주의사항
```

---

## 새 커맨드 만드는 순서

사용자가 "XXX 커맨드 만들어줘"라고 하면:

### 1단계: 목적 확인

한번에 물어봐:
```
커맨드를 만들기 전에 확인할게요:

1. 이 커맨드는 어떤 상황에서 쓸 건가요?
2. 인자(추가 입력)가 필요한가요? 예: /커맨드 "내용"
3. 결과물을 파일로 저장해야 하나요?
```

### 2단계: 기존 커맨드 확인

```
ls .claude/commands/
```
비슷한 커맨드가 있으면 알려주고 수정할지 새로 만들지 물어봐.

### 3단계: 커맨드 파일 생성

저장 위치: `.claude/commands/커맨드이름.md`
- 이름: 소문자 + 하이픈 (예: `weekly-review.md`)
- 내용: 위 형식 참고

### 4단계: 완료 알림

```
✅ '/커맨드이름' 커맨드를 만들었어요!

사용 방법: /커맨드이름 입력하면 바로 동작해요.
파일 위치: .claude/commands/커맨드이름.md

지금 테스트해볼까요?
```

---

## 좋은 커맨드의 특징

- **한 가지 일만** 한다
- **실행 순서가 명확**하다 (Claude가 헷갈리지 않게)
- **저장 경로가 명시**되어 있다 (파일을 만들 경우)
- **짧고 명확**하다 (200줄 이하 권장)
