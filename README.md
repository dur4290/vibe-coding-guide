# 바이브코딩 워크스페이스

AI와 함께 코딩하는 개인 작업 공간입니다.  
Claude가 이 폴더를 읽고 여러분의 자동화 파트너가 됩니다.

---

## 자주 쓰는 흐름

처음 열었다면 Claude Code 입력창에 `/start`를 입력하세요.

아이디어가 떠올랐다면 `/idea 아이디어 내용`으로 메모하고, 실제로 만들고 싶어지면 `/clarify 아이디어 내용`으로 계획서를 만드세요.

작업을 끝냈거나 중간 저장하고 싶다면 `/save 저장 메시지`를 입력하세요.

가이드북을 끝낸 뒤 다음에 뭘 만들지 고민된다면 `learning-path.md`를 열어보세요. 자동화, 문서/발표, 웹앱, 나만의 작업 시스템으로 이어지는 다음 프로젝트 예시가 들어 있습니다.

```text
아이디어 메모 → 계획서 만들기 → projects/에 프로젝트 생성 → 실행하고 수정 → /daily 기록 → /save 저장
```

막히면 새 커맨드를 찾기보다 Claude에게 이렇게 말하세요.

```text
현재 워크스페이스 상태를 확인하고, 내가 다음에 뭘 해야 하는지 알려줘.
```

## 시작하기

> **필수 준비물**: Claude Pro 구독 (월 ₩30,000 / claude.ai) + VSCode + Claude 확장

1. VSCode에서 이 폴더 열기
2. 왼쪽 사이드바에서 Claude 아이콘 클릭
3. `/start` 입력 — 나머지는 Claude가 안내해줍니다

처음 시작하면 Claude가 자동으로 환경을 설정하고 질문지를 띄워줘요.

> 팁: 터미널 명령을 직접 외울 필요는 거의 없습니다. 필요한 확인과 실행은 Claude가 먼저 처리하고, 여러분은 결과를 보고 다음 결정을 하면 됩니다.

## 슬래시 커맨드

Claude 채팅창에 `/커맨드` 를 입력하면 동작합니다.

| 커맨드 | 설명 | 저장 위치 |
|--------|------|-----------|
| `/start` | 처음 시작할 때 — 내 정보 입력 → 첫 프로젝트 추천 | `CLAUDE.md` 업데이트 |
| `/clarify [아이디어]` | 막연한 아이디어 → 구체적인 실행 계획서 | `notes/tasks/` |
| `/idea [아이디어]` | 아이디어 빠르게 메모 | `notes/ideas/` |
| `/daily` | 오늘 배운 것 기록 (5개 질문) | `notes/daily/` |
| `/save` | GitHub에 저장 (보안 확인 → 커밋 → push) | GitHub |

### 커맨드 사용 예시

```
/clarify 매일 학교 공지사항 확인하는 게 너무 귀찮아
/idea 유튜브 자막 자동 저장 스크립트
/daily
/save 파일 정리 자동화 완성
```

## 스킬

Claude가 특정 작업을 요청받으면 자동으로 불러오는 도구입니다.  
자연어로 말하면 돼요 — 커맨드 따로 입력할 필요 없어요.

| 스킬 | 트리거 문구 | 하는 일 |
|------|------------|---------|
| `web-scraper` | "이 사이트 긁어줘", "웹에서 데이터 가져와줘" | robots.txt 확인 후 BeautifulSoup/Playwright 스크립트 자동 생성 |
| `pdf` | "PDF 읽어줘", "PDF 내용 요약해줘" | PDF에서 텍스트·표 추출, 요약 |
| `skill-creator` | "새 스킬 만들어줘", "XXX 스킬 만들어줘" | 올바른 형식으로 새 스킬 생성 |
| `slash-command-creator` | "새 커맨드 만들어줘", "/XXX 커맨드 만들어줘" | 올바른 형식으로 새 슬래시 커맨드 생성 |
| `subagent-creator` | "새 에이전트 만들어줘", "XXX 에이전트 만들어줘" | 올바른 형식으로 새 에이전트 생성 |

## 에이전트

특정 상황이 되면 Claude가 자동으로 도와주는 전문가입니다.

| 에이전트 | 자동 실행 조건 | 하는 일 |
|----------|--------------|---------|
| `code-simplifier` | 새 코드가 만들어졌을 때, "코드 이해 안 가"라고 할 때 | 한국어 주석 추가, 변수명 개선, 코드 정리 |
| `markdown-optimizer` | SKILL.md나 커맨드 파일이 500줄 넘을 때, "파일이 너무 길어"라고 할 때 | 내용은 유지하면서 토큰 절약 (산문→불릿, 중복 제거) |

    ├── commands/      ← 슬래시 커맨드 파일들
    ├── skills/        ← 스킬 파일들
    └── agents/        ← 에이전트 파일들

## 자동 설정

이 워크스페이스는 VSCode에서 열고 Claude와 대화를 시작하면 자동으로 설정돼요.

- `.setup-done` 파일이 없으면 → `/start` 온보딩 시작
- `.setup-done` 파일이 있으면 → 바로 작업 시작
- GitHub 저장소와 연결된 뒤에는 `/save`가 보안 훅을 자동으로 활성화합니다
- Python이나 Anaconda가 이미 설치되어 있으면 Claude가 경로를 찾아서 이 워크스페이스에서 사용할 수 있게 기억합니다

별도로 PowerShell이나 터미널 설정이 필요하지 않습니다.

## 자동 저장 위치

Claude에게 보고서나 조사 문서를 만들어달라고 하면 기본적으로 `notes/reports/` 폴더에 마크다운 파일로 저장됩니다.

예를 들어 “인기 AI 도구를 조사해서 마크다운 보고서를 만들어줘”라고 요청하면, Claude가 `notes/reports/2026-05-18-ai-tools-report.md` 같은 파일을 자동으로 만듭니다.

단일 자동화 스크립트는 기본적으로 `scripts/` 폴더에 저장됩니다.
예를 들어 파일 정리 자동화는 `scripts/organize.py`처럼 저장하면 됩니다.

## 보안

`/save` 커맨드는 저장 전 자동으로 보안 검사를 합니다.

- `.env` 파일이 포함되어 있는지 확인
- API 키 패턴(`sk-ant-`, `sk-proj-` 등)이 코드에 없는지 확인

이 수업의 기본 실습에서는 API 키를 직접 발급받거나 코드에 넣지 않습니다.  
요약이나 문장 정리는 Claude Code 안에서 처리하고, Python 스크립트는 파일 정리·텍스트 추출·CSV 처리처럼 내 컴퓨터의 로컬 작업을 담당합니다.

```python
# 위험 (API 키 직접 작성)
api_key = "실제_API_키_문자열"

# 기본 실습 권장
# Python: 로컬 파일 처리
# Claude Code: 요약, 분석, 보고서 작성
```

외부 API를 쓰는 별도 앱을 만들고 싶다면, 그때 Claude에게 비용과 보안 주의사항을 먼저 설명해달라고 요청하세요.

## 가이드북 이후

이 워크스페이스는 가이드북 실습이 끝나도 계속 사용할 수 있습니다.

- 새 자동화 아이디어는 `notes/ideas/`에 모읍니다.
- 만들기로 결정한 아이디어는 `/clarify`로 `notes/tasks/`에 계획서로 남깁니다.
- 실제 작업은 `projects/프로젝트이름/` 안에서 진행합니다.
- 완성된 단일 스크립트는 `scripts/`에 정리할 수 있습니다.
- 배운 것과 막힌 점은 `/daily`로 기록합니다.
- 중요한 변경은 `/save`로 GitHub에 저장합니다.

다음 단계가 막연하면 Claude에게 이렇게 말해보세요.

```text
learning-path.md를 참고해서 지금 내 수준에서 해볼 만한 다음 프로젝트를 추천해줘.
```
